### 1. BUG 8 — Summon không stack-based (đã fix)

**File:** `AbilityEngine.cs` — `ExecuteEffect > case EffectType.Summon`

**Vấn đề:**
- `EffectType.Summon` với `effectValue1 = 2` chỉ summon 1 unit (bỏ qua effectValue1).
- Sau đó fix bằng vòng lặp đơn giản, nhưng tất cả unit được summon liên tiếp ngay lập tức → Sekhmet có thể consume cả NW1 lẫn NW2 trong cùng một lần broadcast trước khi death chain của NW1 (→ Mummy) kịp resolve.

**Fix cuối cùng (Stack-based Summon):**
- Unit đầu tiên (NW1) được summon và broadcast **ngay lập tức**.
- Các unit còn lại (NW2, NW3...) đi vào `_pendingSummons` queue.
- `CombatResolver.FlushDeathStack` chỉ pop 1 pending summon **sau khi** death stack hoàn toàn rỗng — tức là sau khi toàn bộ death chain của NW1 (bao gồm Mummy, OnAllyDeath, Reborn...) resolve xong.
- Nếu không còn slot → `SummonUnit` trả `null`, unit biến mất theo đúng luật.

**Files thay đổi:**
| File | Thay đổi |
|------|----------|
| `AbilityEngine.cs` | Thêm `PendingSummonEntry` struct, `_pendingSummons` queue, `HasPendingSummons`, `ClearPendingSummons()`, `ProcessNextPendingSummon()` |
| `CombatResolver.cs` | `ResolveTurn`: thêm `engine.ClearPendingSummons()`; `FlushDeathStack`: sau CleanupBoard, nếu stack rỗng thì pop 1 pending, scan deaths, lặp lại |

### 2. BUG 9 — Hai Sekhmet: SekhmetB tốn triggerLimit count khi target đã chết (đã fix)

**Files:** `AbilityEngine.cs` + `AbilityEngine.Targets.cs`

**Nguyên nhân (2 lỗi cộng hưởng):**
1. **`TriggerSubject` không check `IsDead`**: Khi SekhmetA consume SandGolem xong, SekhmetB vẫn nhận `SandGolem` trong danh sách target (dù đã chết).
2. **`abilityTriggerCounts[i]++` trước `FindTargets`**: Count bị tăng trước khi biết target có hợp lệ không. `ExecuteEffect` early-return vì `target.IsDead` → count bị lãng phí.

**Kết quả bug:** SekhB kết thúc với `count=2` (đã cạn limit) nhưng `consumedCardIDs=[]`. Khi chết không release gì cả.

**Fix 1 — `AbilityEngine.Targets.cs:87`:**
```csharp
case TargetType.TriggerSubject:
    // BUG FIX: Thêm !directEnemy.IsDead
    if (directEnemy != null && !directEnemy.IsDead) validTargets.Add(directEnemy);
    break;
```

**Fix 2 — `AbilityEngine.cs` — `TriggerAbility`:**
- Với ability có `triggerLimit > 0`: `FindTargets` chạy **trước** count++. Nếu `targets.Count == 0` → `continue` không tốn count.
- Với `conditionCount` (Bastet, NilesCultist): vẫn luôn đếm mọi trigger, chỉ execute vào đúng lần N.

```csharp
List<CardInstance> targets = FindTargets(ability, source, directEnemy, allyBoard, enemyBoard);

if (ability.triggerLimit > 0)
{
    if (targets.Count == 0) continue;   // không tốn count khi target dead
    source.abilityTriggerCounts[i]++;
}
else
{
    source.abilityTriggerCounts[i]++;
    if (ability.conditionCount > 0 && source.abilityTriggerCounts[i] % ability.conditionCount != 0) continue;
}
```
