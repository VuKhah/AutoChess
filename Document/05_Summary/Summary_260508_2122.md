## 3. Refactor: ability → abilities (List\<AbilityData\>)

**Lý do:** `CardDefinition.ability` là single object nên mỗi unit chỉ có thể có 1 ability. Game AutoChess cần unit với combo ability (Deploy + Deathrattle, Aura + OnAttack, v.v.).

**Phạm vi thay đổi:**

### CardDefinition.cs
```
AbilityData ability  →  List<AbilityData> abilities
```

### CardInstance.cs
```
int abilityTriggerCount  →  List<int> abilityTriggerCounts
```
- `ResetStats()`: tái tạo list `new int[abilities.Count]` thay vì reset 1 số nguyên.
- Constructor: `isTaunt` check qua `abilities.Exists(a => a.isTaunt)`.
- `ApplyMagicToUnit` sau khi thêm ability mới gọi `ResetStats()` → list tự resize đúng.

### CombatResolver.cs — TriggerAbility()
Thay vì check 1 ability, giờ loop qua toàn bộ list với index `i`:
- Mỗi ability có counter riêng `abilityTriggerCounts[i]`.
- Nhiều ability cùng trigger context đều được thực thi trong cùng 1 lần gọi.

### CombatResolver.cs — ExecuteEffect() case TriggerAbility
Loop qua `target.Data.abilities`, fire từng ability (guard chống chain vô hạn giữ nguyên).

### CombatResolver.cs — ApplyMagicToUnit() case AddAbility
Từ gán đè `unit.Data.ability = magic.Data.ability` → **append** toàn bộ abilities của magic vào list của unit.  
Unit vẫn giữ abilities cũ, magic chỉ bổ sung thêm.

### GameManager.cs — EndTurnShop coins
Đổi từ check 1 ability sang loop qua `unit.Data.abilities` để tổng hợp tất cả GainCoin abilities trong 1 unit.

### CardUI.cs
`firstActive = abilities.Find(a => a.trigger != None)` — lấy ability đầu tiên có trigger để hiển thị icon. Taunt vẫn ưu tiên icon 99.

### BotAgent.cs — ScoreCard()
Đổi 5 điều kiện if sang `c.abilities.Exists(a => ...)`.

### CardsData.json
47 entries đổi từ `"ability": {...}` → `"abilities": [{...}]` bằng regex.  
Tất cả cards giờ dùng array — thêm ability thứ 2 chỉ cần append object vào mảng JSON.
